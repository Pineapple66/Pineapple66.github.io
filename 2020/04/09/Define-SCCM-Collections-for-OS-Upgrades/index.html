Define SCCM Collections for OS Upgrades

Posted on 2020-04-09 Edited on 2021-03-09 In System Administration

Collections help you organize resources into manageable units. You can create collections to match your client management needs, and to perform operations on multiple resources at one time. In order to perform the OS upgrades for all targeted clients, collection would be the best way to locate them.

This tutorial will show you how to locate any active but out-of-date clients by using the OS version number.

Task: locate all Windows 10 clients have version 1803 and/or eariler.

Step 1 Create a new Collection in SCCM

Navigate to the correct collection and right click on any empty area

Step 2 Define the Scope

Select the correct collection for locating these PCs, they are all under “ALL Workstations”. In some of cases, you may do “ALL xx location workstations” OR “All servers”.

Step 3 Define the Membership Rules

Follow the steps icon, you will need to locate the Operating system version number.

Step 4 Define the Operating System number

Windows 10 version 1803 has built number 17134, you may locate some common numbers on this table.

Windows 10 (20H2) 10.0.19042
Windows 10 (2004) 10.0.19041
Windows 10 (1909) 10.0.18363
Windows 10 (1903) 10.0.18362
Windows 10 (1809) 10.0.17763
Windows 10 (1803) 10.0.17134
Windows 10 (1709) 10.0.16299
Windows 10 (1703) 10.0.15063
Windows 10 (1607) 10.0.14393
Windows 10 (1511) 10.0.10586
Windows 10 10.0.10240

Step 5 Schedule the Membership updates

Default value is used on this task, 7 days is fine, but you may change to sooner.

Step 6 Confirm and finish

You are Done!

At this point, you have successfully created the collection, you may need to wait around 30 mins before seen any members.
