Using SCCM to Perform Zoom Upgrades

Posted on 2020-08-15 Edited on 2021-03-09 In System Administration

Zoom has became our friend since 2015. In order to fully take advantage of the new features that zoom has offered, I needed to keep zoom clients updated on all computers campus wide. This is a comprehensive guide to show you how it is done!

Task: Target all workstations and perform the silence upgrade

Step 1 Locate and Download the MSI file

Navigate to https://zoom.us/download, and download the MSI for IT admin

Step 2 Move the files to Network location

Launch Explorer++ with network level administrative account, create a new folder under Zoom and rename it to the current release version, and copy the file that you just download from Zoom.us to this folder.

Step 3 Create a new application on SCCM management console.

Navigate to \Software Library\Overview\Application Management\Applications\Zoom, right click on any of the empty area, and select create application.

Step 4 Select the correct directory

Select the path for installation package (s), click next when you are done.

Step 5 Specify the name and version information

For name, software version number is being added at the end of the line. Select install for system, and following string is used for installing:

Click next when we done.

Step 6 Add an icon to this software

With this program selected, right click, and select properties. With the Zoom 5.4.0 Properties window open, select “Software Center” tab. Locate the correct icon and hit open.

Step 7 Uninstall Previous Version

With the Zoom 5.4.0 Properties window open, select “Supersedence” tab. Under the “New Deployment Type”, Select the “Zoom- Windows Installer (*.msi file)”, and check the “Uninstall” button at the end of the column.

Repeat these steps until all previous versions are selected.

Step 8 Distribute content and deploy the software

Software must distributed before deploy.

Right click this software again and select “Deploy”

Select the desire distribution point or group

Step 9 Select the installation behavior and Time

In this test deploy, “available” is used for demo.

Step 10 Job Done.

At this point, software updates has been scheduled. All clients will receive the new software notices and apply the updates accordingly.
